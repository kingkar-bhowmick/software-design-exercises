Aufgabe 1: 

Frage 1: Worin besteht der Unterschied zwischen einem Fork und einem Clone?
Antwort: Ein Fork ist eine Funktion in GitLab (oder GitHub), mit der man eine Kopie eines Repositorys eines anderen Benutzers auf dem eigenen Account erstellt. Ein Clone hingegen ist ein Git-Kommando, mit dem man ein Repository (egal ob geforkt oder nicht) lokal auf dem eigenen Computer kopiert, um daran zu arbeiten.

2. Warum ist der Fork für Teamarbeit sinnvoll?
Antwort: Ein Fork ist für die Teamarbeit sinnvoll, weil er es ermöglicht, Änderungen an einer Kopie des ursprünglichen Repositorys vorzunehmen, ohne den Main Branch des Originalprojekts zu beeinflussen.
Teammitglieder können unabhängig voneinander an ihren Forks arbeiten und anschließend über einen Merge Request (oder Pull Request) ihre Änderungen zur Überprüfung und Integration in das Hauptrepository vorschlagen.

3. Wer kann in Ihrem Fork Änderung machen?
Antwort: Änderungen in einem Fork können nur vom Besitzer (Owner) des Forks und von Mitgliedern mit entsprechenden Berechtigungen (z. B. Developer-Rolle) vorgenommen werden.
Das bedeutet, dass nur die Personen, denen Bearbeitungsrechte für den Fork erteilt wurden, Änderungen durchführen können.
